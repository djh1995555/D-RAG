## 一、产品概述

### 1.1 产品定位

城市猎人是一款基于地理位置的城市解谜小程序。玩家通过对话形式推进剧情，在对话中获取线索，线索指向真实的城市地点。玩家前往实地探索地标和文化景点，通过现场寻找线索解开谜题，最终完成整个剧情。

### 1.2 产品愿景

让每个城市都成为一个巨大的解谜游戏场，让玩家在探索中发现城市的故事。

### 1.3 核心特色

| 特色 | 说明 |
|------|------|
| 实地探索 | 结合真实城市地标，增强沉浸感 |
| 剧情驱动 | 分支对话 + 多重结局，重玩价值高 |
| 实体道具 | 部分案件配套实物道具，增强代入感 |
| 线索推理 | 地点不直接展示，玩家需从对话中推理 |

### 1.4 目标平台

| 阶段 | 平台 | 说明 |
|------|------|------|
| 一期 | 微信小程序 | 验证核心玩法，快速获客 |
| 二期 | 独立 App | 扩展功能，提升体验 |

### 1.5 首发策略

先在上海深耕，打磨产品体验，验证商业模式，后期扩张至其他城市。

## 二、市场分析

### 2.1 市场背景

#### 2.1.1 行业趋势

| 趋势 | 说明 |
|------|------|
| 体验经济崛起 | 消费者越来越重视体验而非物质消费 |
| 城市微度假 | 疫情后短途、本地游成为主流 |
| 沉浸式娱乐 | 剧本杀、密室逃脱市场快速增长 |
| 文旅融合 | 文化内容与旅游深度结合 |

#### 2.1.2 市场规模

| 市场领域 | 2024年规模 | 年增长率 |
|----------|------------|----------|
| 剧本杀市场 | 约200亿元 | 30%+ |
| 城市徒步/探索 | 约50亿元 | 40%+ |
| 文旅融合市场 | 约1000亿元 | 20%+ |

### 2.2 竞品分析

#### 2.2.1 直接竞品

| 产品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| 某城市解谜App | 城市探索解谜 | 内容丰富，多城市覆盖 | App下载门槛高 |
| 某剧本杀小程序 | 线上剧本杀 | 用户量大 | 无实地探索体验 |
| 某徒步App | 城市徒步路线 | 路线丰富 | 无剧情驱动 |

#### 2.2.2 间接竞品

| 产品 | 定位 | 说明 |
|------|------|------|
| 剧本杀门店 | 线下沉浸式娱乐 | 高客单价，需预约 |
| 密室逃脱 | 线下解谜体验 | 场地限制，需团队 |
| 城市漫步活动 | 线下导览活动 | 固定时间，需报名 |

#### 2.2.3 差异化优势

| 维度 | 城市猎人 | 竞品 |
|------|----------|------|
| 获客门槛 | 小程序即开即玩 | App需下载 |
| 体验形式 | 实地探索+剧情 | 纯线上或纯线下 |
| 时间灵活性 | 随时开始随时暂停 | 固定时段 |
| 价格带 | ¥39-59/案件 | ¥100-300/次 |
| 社交属性 | 可独行可组队 | 多需组队 |

### 2.3 市场机会

1. **填补空白**：结合线上剧情+线下探索的产品较少
2. **低门槛获客**：小程序无需下载，降低用户尝试成本
3. **城市IP价值**：每个城市都可以开发专属内容
4. **文旅合作空间**：可与景区、博物馆、商业体合作

### 3.2 用户旅程地图

#### 首次使用旅程

```
发现阶段
    ↓
看到朋友分享/公众号推荐/朋友圈广告
    ↓
兴趣阶段
    ↓
点击进入小程序 → 浏览首页 → 查看案件详情
    ↓
决策阶段
    ↓
选择案件 → 查看价格 → 决定购买
    ↓
体验阶段
    ↓
开始游戏 → 边走边玩 → 完成案件
    ↓
分享阶段
    ↓
达成结局 → 分享结果 → 推荐给朋友
```

#### 复购旅程

```
提醒阶段
    ↓
收到新案件推送/朋友邀请
    ↓
回访阶段
    ↓
打开小程序 → 查看新案件 → 购买体验
    ↓
口碑传播
    ↓
主动分享 → 邀请朋友一起玩
```

## 六、数据埋点方案

### 6.1 核心指标体系

#### 6.1.1 北极星指标

| 指标 | 定义 | 目标 |
|------|------|------|
| 月活跃玩家数 | 每月至少完成一个案件的独立用户数 | 持续增长 |

#### 6.1.2 一级指标

| 指标 | 定义 | 说明 |
|------|------|------|
| DAU | 日活跃用户数 | 用户活跃度 |
| 付费转化率 | 付费用户数 / 活跃用户数 | 商业化能力 |
| 案件完成率 | 完成案件数 / 开始案件数 | 内容吸引力 |
| 用户留存率 | 次日/7日/30日留存 | 产品粘性 |

#### 6.1.3 二级指标

| 指标 | 定义 | 说明 |
|------|------|------|
| 人均案件数 | 总开始案件数 / 总用户数 | 用户参与深度 |
| 人均时长 | 总游戏时长 / 总用户数 | 用户投入程度 |
| 提示使用率 | 使用提示次数 / 总游戏次数 | 难度是否合理 |
| 结局分布 | 各结局达成比例 | 内容设计合理性 |
| 分享率 | 分享用户数 / 完成用户数 | 口碑传播 |

## 十、核心业务流程

### 10.1 案件购买流程

```
用户选择案件 → 检查购买状态
    ↓
未购买：检查是否有实体道具
    ├── 有：引导填写收货地址 → 发起支付
    └── 无：直接发起支付
    ↓
支付成功 → 创建进度记录（current_node_id = start_node_id）
    ↓
允许开始游戏
```

### 10.2 对话推进流程

```
展示当前节点内容
    ↓
玩家可随时长按对话 → 收藏到笔记本
    ↓
判断节点类型
    ├── 普通对话：自动推进到下一节点
    ├── 选择分支：显示选项，记录选择，跳转目标节点
    ├── 地点打卡：进入打卡流程
    ├── 答案验证：进入验证流程
    └── 结局节点：计算结局，结束游戏
```

### 10.3 分支选择流程

```
当前节点有分支选项
    ↓
检查每个选项的前置条件
    ├── 条件满足：显示该选项
    └── 条件不满足：隐藏该选项
    ↓
玩家从可见选项中选择
    ↓
记录分支选择到 branch_state
    ↓
跳转到 next_node_id
```

### 10.4 地点打卡流程

```
对话推进到 location_check 节点
    ↓
展示线索提示（无地点名称）
    ↓
玩家线下探索，推理目标地点
    ↓
玩家到达后点击"验证地点"
    ↓
获取当前GPS坐标
    ↓
遍历该案件所有 locations，计算距离
    ↓
    ├── 找到匹配地点（距离≤50米）：打卡成功，展示地点详情
    └── 无匹配地点：提示"这里似乎不是目标地点"
    ↓
更新进度，解锁后续对话
```

### 10.5 答案验证流程

```
对话推进到 answer_verify 节点
    ↓
展示问题和答案来源提示
    ↓
玩家输入答案
    ├── 正确：验证成功，继续剧情
    └── 错误：错误计数+1
        ├── 次数<3：提示重试
        └── 次数≥3：提供额外提示
```

### 10.6 提示系统流程

```
对话界面显示"提示"按钮（显示剩余次数）
    ↓
玩家点击提示 → 弹窗确认
    ↓
确认后扣减提示次数
    ↓
根据当前节点类型返回对应提示
    ├── 普通对话：提示后续剧情方向
    ├── 地点打卡：提示地点方向/区域
    └── 答案验证：提示答案相关线索
```

### 10.7 结局计算流程

```
对话推进到结局节点
    ↓
读取 user_progress.branch_state
    ↓
查询该案件所有 endings
    ↓
按 conditions 匹配
    ├── 匹配成功：达成特定结局
    └── 无匹配：达成默认（普通）结局
    ↓
更新进度（is_completed=true, ending_id）
    ↓
展示结局画面 + 游戏统计 + 成就
```

### 10.8 线索收藏流程

```
对话界面展示
    ↓
玩家长按对话消息
    ↓
进入多选模式（类似微信）
    ↓
勾选一条或多条对话
    ↓
点击"加入笔记本"
    ↓
可添加个人标注
    ↓
保存到 user_notes
```
