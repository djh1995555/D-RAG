# **GSD2工作流**
1. 用`gsd2`框架
2. 用`superpower`的`brainstorming`来完善需求文档，然后加上我认为系统应该知道的上下文
3. 让gsd分析requirment.md，让他分析有哪些内容需要放进KNOWLEDGE
4. 进入`gsd2`工作流，一开始输入`/gsd`，然后系统会问第一个里程碑，这时候让他读需求文档，就可以开始了。把完整UI设计成第一个里程碑
5. 计划文档review，重点看UI布局，需求点是否覆盖全，单元测试方案，**端到端阶段验收方案**，没有的话要让他们加上
6. `/gsd auto`开启第一个里程碑UI设计开发，设计结束开始优化
7. 用impeccable来优化UI，`/teach-impeccable `，然后循环优化，`循环运行audit和它给出的建议命令，直到audit给出的分数不再变化`
8. `/gsd auto`继续后面的开发

# **superpower工作流**
1. 给出requirment.md，让claude分析，生成CLAUDE.MD
2.  /writing-plans -> /executing-plans -> /test-driven-development

### Core Analysis Flags
| 标志  | 激活条件 | 功能  |
| --- | ---- | --- |
