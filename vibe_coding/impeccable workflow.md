
## 📋 前置准备（必做）

- [ ] 确认项目已接入支持Impeccable的AI助手，可正常调用斜杠命令

- [ ] 准备项目核心资料：品牌配色、字体规范、设计调性、目标设备、无障碍要求

- [ ] 新建 `.impeccable.md` 文件，用于存储设计规范与优化全程记录

---
## 🔄 全流程优化步骤（按顺序执行）

### 第一阶段：项目设计规范初始化 ✅

**核心目标**：让AI明确项目设计规则，避免后期优化跑偏

| 项目 | 详情 
| :--- | :--- |
| 执行命令 | `/teach-impeccable` |
| 操作要点 | 1. 允许AI扫描项目README、现有组件、样式文件、<br>2. 逐一回复AI提问，明确设计规则、禁忌、<br>3. 将AI生成的规范同步写入 `.impeccable.md` 留存 |
| 验收标准 | ✅ 完成项目专属设计上下文搭建，AI明确设计底线与风格 |

---
### 第二阶段：UI问题全面诊断审计 ✅

**核心目标**：系统性找出所有UI/UX缺陷，不盲目优化

| 项目 | 详情 |
| :--- | :--- |
| 必执行命令 | `/audit`（局部优化：`/audit 具体模块`，如 `/audit header`） |
| 可选补充命令 | `/critique`（UX评审专用） |
| 操作要点 | 1. 运行 `/audit`，获取P0-P3级问题清单（阻塞性+优化性）<br>2. 优先处理P0/P1级问题，标记P2<br>3. （可选）运行 `/critique`，评审视觉层级、用户<br>4. 整理问题清单，明确优化优先级 |
| 验收标准 | ✅ 输出完整问题报告，无核心缺陷遗漏，优先级清晰 |

---

### 第三阶段：设计规范统一与健壮性优化 ✅

**核心目标**：消除样式混乱，补全组件边界场景

| 项目 | 详情 |
| :--- | :--- |
| 必执行命令 | `/normalize` → `/harden` |
| 可选补充命令 | `/distill`（精简冗余） |
| 操作要点 | 1. `/normalize`：统一间距、字体、颜色、圆角、阴影等<br>2. `/harden`：补全错误态、空状态、溢出、多语言、<br>3. （可选）`/distill`：移除冗余嵌套、无意义装饰、过度特效 |
| 验收标准 | ✅ 样式规范统一，组件覆盖所有使用场景，无交互漏洞 |

---

### 第四阶段：视觉质感专业精修 ✅

**核心目标**：摆脱“AI味”，提升UI专业度

| 项目 | 详情 |
| :--- | :--- |
| 执行命令 | `/typeset` → `/colorize` → `/arrange` |
| 操作要点 | 1. `/typeset`：优化字体配对、行高、字重层级、响应式<br>2. `/colorize`：应用OKLCH色彩，优化对比度<br>3. `/arrange`：调整网格对齐、留白、视觉流、组件间距 |
| 验收标准 | ✅ 排版清晰、色彩合规、布局规整，无低质设计反模式 |

---

### 第五阶段：交互与动效优化 🔘（推荐）

**核心目标**：让交互更流畅，增加用户体验细节

| 项目 | 详情 |
| :--- | :--- |
| 执行命令 | `/animate` → `/delight` |
| 操作要点 | 1. `/animate`：添加专业缓动，开启“减少动效”适配<br>2. `/delight`：优化微交互，补充按钮、输入框等状态反馈 |
| 验收标准 | ✅ 动效有意义、不炫技，交互反馈及时，支持动效偏好设置 |

---
  
### 第六阶段：性能与全设备适配 ✅

**核心目标**：确保上线后快、稳、全设备兼容

| 项目 | 详情 |
| :--- | :--- |
| 执行命令 | `/optimize` → `/adapt` |
| 操作要点 | 1. `/optimize`：精简CSS/JS、优化渲染<br>2. `/adapt`：移动优先适配，优化平板、大屏、容器查询效果 |
| 验收标准 | ✅ 性能提升，全设备展示正常，无适配BUG |

---

### 第七阶段：最终收尾与规范沉淀 ✅

**核心目标**：零瑕疵上线，沉淀可复用规范

| 项目 | 详情 |
| :--- | :--- |
| 必执行命令 | `/polish` |
| 可选补充命令 | `/extract`（提取规范） |
| 操作要点 | 1. `/polish`：像素级细节优化，检查全<br>2. （可选）`/extract`：提取设计token、组件规范，更新样式<br>3. 再次运行 `/audit` 复检，确认无遗留问题 |
| 验收标准 | ✅ UI无细节瑕疵，符合所有规范，可直接上线 |
